# 需求分析

目标是在 `D:\Documents\C_learn\成绩查询\tauri-app` 项目内继续执行上一轮小游戏优化计划，并扩大到新增棋类小游戏与多人在线对战能力。核心范围包括：

1. 修复 `website/modules-src/clumsy_bird_hbut` 在 Tauri/模块宿主中出现页面过小、双重缩放的问题，统一为竖屏满屏可玩页面。
2. 大幅重做 `hbut_monopoly`、`hbut_memory_match`、`hbut_miner` 的玩法深度、挑战性和移动端页面表现。
3. 在可行范围内新增棋类小游戏，并调研公开可用的多人在线方案，优先选择公开服务或开源方案，保证移动端美观可用。

# 当前上下文

- 项目根目录已有 `goal-1`、`goal-2`，本目标使用新的 `goal-3`，不得覆盖已有目录。
- 当前分支是 `main`，仓库已有未提交改动，主要集中在 `src/*`、`website/public/modules/*` 和 `data/`。本目标实施时不得回退这些改动。
- 现有小游戏源码集中在 `website/modules-src/*/project`。
- 多数小游戏是 Vite 单页模块，入口由各自 `module.json` 声明，子页面通过 `mini-hbut:module-size` 向宿主上报高度。
- 当前四个重点小游戏均没有独立测试脚本；需要先补测试依赖和核心逻辑测试，再改实现。

# 风险

- 玩法重做涉及多个小游戏，容易引入回归；必须先对核心规则补单元测试，再实现。
- Tauri 宿主与子模块都有视口和高度同步逻辑，双重缩放需要通过实际移动端视口验收确认。
- 多人在线对战依赖公开服务时，外部服务可用性、免费额度、隐私和生产稳定性不可完全控制；计划中必须明确默认方案、降级方案和不触碰敏感生产配置。
- 新增棋类小游戏范围较大，应先做一个可独立运行、可本地/在线扩展的棋类模块，再逐步扩展多人功能。

# 执行方案

1. 先完成目标初始化文件，不改代码。
2. 每轮只执行 `tasks.md` 中第一个未完成任务。
3. 每个代码任务遵循测试先行：先写失败测试，确认失败，再写实现，再验证通过。
4. 每完成三个任务执行一次大型全面检查-debug 循环，检查需求偏离、构建、测试、UI、数据一致性、安全和回滚。
5. 多人在线方案先进行公开服务调研，再选择最小可落地架构。默认优先评估：
   - WebRTC/P2P 信令服务：PeerJS Cloud、Trystero 支持的公共信令后端、Nostr relay 等公开方案。
   - 托管实时数据库/消息服务：Supabase Realtime、Firebase/Firestore、Ably、Pusher、PartyKit 等。
   - 开源自托管方案：Colyseus、boardgame.io、Nakama，作为可部署选项，不默认改生产配置。
6. 新增棋类小游戏默认先做“湖工五子棋”：规则简单、移动端适配清晰、多人同步模型小，后续可扩展象棋/围棋/黑白棋。

# 验证方式

- 单元测试：
  - 矿工：关卡初始化、钩长覆盖最深物品、底部物品可命中、道具效果、过关推进。
  - 记忆牌：关卡初始化、倒计时失败、评分、连击、错配惩罚、预览逻辑。
  - 大富翁：事件选择、资源边界、卡牌效果、阶段胜负、骰子合法性。
  - 棋类：落子合法性、胜负判断、悔棋/重开状态、在线消息幂等。
- 构建：
  - 每个改动模块运行 `npm run build`。
  - 必要时运行根目录 `npm run build` 或相关 Vitest。
- UI 验收：
  - 使用浏览器在 360x740、390x844、430x932 竖屏视口检查主要页面。
  - 验证无横向滚动、核心按钮可触达、文本不溢出、游戏主体不被压缩过小。
- 在线对战：
  - 验证本地双标签页或双浏览器房间同步。
  - 外部服务不可用时应显示可理解错误，并保留本地单机模式。

# 回滚方案

- 每个任务结束后提交独立 git commit；若出现高风险问题，可按任务 commit 回滚。
- 不修改生产密钥、生产环境变量、支付、认证和服务端权限配置。
- 多人在线服务接入使用模块内配置和公开匿名房间协议，默认不开启任何敏感数据上传。
- 若外部服务不稳定，保留棋类单机模式和本地双人同屏模式。

# 默认假设

- 用户选择了“大幅重做”，因此允许重构小游戏内部核心逻辑和 UI，但不改变模块发布协议。
- 新增棋类游戏优先选择五子棋作为首个落地目标。
- 多人在线优先选择免费/公开可试用、浏览器端可接入、移动端友好的方案；若需要账号或密钥，则仅写入可配置接口，不提交真实密钥。
- 所有沟通、记录和关键注释使用简体中文。
