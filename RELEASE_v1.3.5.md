# Mini-HBUT v1.3.5 Release Notes

## 📢 发版概述
本次更新（v1.3.4 -> v1.3.5）是一次架构层面的关键性演进。我们引入了全新的**微前端与动态模块体系**，对「更多」功能页进行了全量解耦与模块化重构。同时，本次更新大幅度增强了 CI/CD 构建流水线的健壮性，优化了 GitHub Release 发布通道，并修复了众多边缘场景下的静态资源加载与原生桥接异常问题。

---

## ✨ 架构演进与模块化 (Architecture & Modularization)
- **动态模块加载底座**：重构了「更多」视图（More Page），引入动态 Website Module 加载机制，实现了核心与边缘业务在运行时的彻底解耦。
- **在线学习模块化抽取**：将原本内置的“在线学习（刷课）”功能剥离为独立的 `shuake` 云端功能包，并通过远端配置（`remote-config`）实现下发与管控。
- **合成模块推流**：成功集成并推送基于新架构的「合成分数（Hecheng）」模块进入主通道（Main Channel），并进一步修正了对应的打包构建流（Bundle Flow）与宿主接口逻辑。
- **容器与嵌入性优化**：实现并在主窗口（Main Window）的安全沙箱上下文中直接内嵌云端获取的远程资源模块，提供丝滑如本地应用般的交互一致性。由于采用 Web 视图直接渲染，不仅缩小了应用本体体积，也降低了热更新的链路风险。

---

## 🔧 核心业务与视图修正 (Core Biz & View Fixes)
- **原生文件桥接机制更新**：在获取与读取 `update manifests` 更新清单时，现已全面下放使用 Tauri 原生网桥层（Native Bridge）处理。规避了常规 H5 API 在复杂网络策略下的跨域受限问题。
- **静态资源加载与渲染修正**：彻底排除了电费查询、宿舍列表相关的界面远程静态资源拉取中断及响应挂起等故障。
- **缓存策略时序优化**：修正了模块状态机的初始化管线。确保在调用或者解包前，必须优先触发最新远程模块的拉取行为与视图树合并，拒绝加载过期无效资产。
- **工程构建隔离**：细化了工程内不同微应用打包环节的 `postcss` 工作域，保证微模块的 CSS 不污染或混淆基座工程的通用与全局原子样式。

---

## 🌐 周边生态与官网升级 (Website & Ecosystem)
- **发布页重构与信息同步**：重建了官方发布站点的前端路由与版本渲染逻辑页面，精确恢复并打通了历代历史版本的 Changelog 与下载引流锚点。
- **官网部署数据结构驻留**：在通过统一流向分发最新内容时，全面保护了旧版资产的安全留存状态（Preserve historical website releases），不再因覆盖动作导致历史版本的丢失。

---

## ⚙️ DevOps 与 CI/CD 基础设施 (DevOps & CI/CD)
本次发版期间，我们重整了超过六条不同的自动化工作集，以保障开源版本迭代的标准化：
- **GitHub Release Body 动态截断机制**：修复了极端场景下持续集成输出超过 GitHub 最大载荷体（Body Payload Limit, 125,000 chars）造成的 422 API 重大宕机。目前系统会自动对多数量的单日差异快照执行截断。
- **网站与打包通道一体化（Unify Channels）**：将 Website 与子模块（Module build installs）打包正式编织进单一、安全的 Github Action 循环结构中。涵盖了从构建（Build）、打包（Fallback Zip Packaging）到站点同步（Sync Workflow）的无缝闭环。允许在增量上传阶段实施更为宽容却严谨的安全解覆盖操作（Asset Overwrite）。
- **CI 流水线审计强化**：全方位增强日志审计颗粒度（Harden Workflow Logs），大幅缩短开发者二次接手或处理线上分发故障所需的问题溯源和评估时间。
